## Development Process

### Versioning
We follow [Semantic Versioning](https://semver.org/) (SemVer) for version numbering:
- **MAJOR** version for incompatible API changes
- **MINOR** version for added functionality in a backward-compatible manner
- **PATCH** version for backward-compatible bug fixes

### Release Cycle
- **Monthly** patch releases (0.0.x) for bug fixes
- **Quarterly** feature releases (0.x.0) for new functionality
- **Annual** major releases (x.0.0) for significant updates
